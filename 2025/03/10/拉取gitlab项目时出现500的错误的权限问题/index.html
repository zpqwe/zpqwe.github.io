拉取gitlab项目时出现500的错误的权限问题

Gitlab克隆代码

我遇到的问题错误：

remote: Internal server error
fatal: unable to access The requested URL returned error: 500

问题解决步骤：

1、确定你可以浏览器访问到项目页面

2、确定你的邮箱或账号已添加，有权限可以拉取项目

经过以上2点，我可以确认是我自己本地的问题，通过查询相关资料，发现是ssh密钥没有设置。

解决步骤

本地拉取代码时生成ssh密钥：

你的邮箱：

ssh-keygen -t rsa -C “your.email@example.com” -b 4096

输入命令后，一直按回车，直到出现密钥生成的效果。

密钥查看地址：

C:\Users\zpmei.ssh

打开这个文件，将里面的密钥复制到密钥粘贴框中

复制好之后，点击增加密钥。

2、输入命令，全局设置git本地的账号和邮箱

此步骤完成之后，继续拉取，git clone “Your Git address”。

至此，问题解决，git源码已拉取。
